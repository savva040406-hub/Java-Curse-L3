Структура закодированного файла

1. Заголовок
MAGIC, 4 байта. Используется для проверки формата файла.
originalLength, 4 байта. Хранит размер исходного файла в байтах.

2. Словарь
Далее записывается dictionarySize, 4 байта. Это количество разных байтов в исходном файле.
После этого идут элементы словаря:
value, 1 байт. Исходный байт.
code, строка UTF. Код Хаффмана для этого байта.

3. Данные
После словаря записаны закодированные данные.
Каждый байт исходного файла заменяется своим кодом из словаря.
Запись выполняется по битам. Последний байт при необходимости дополняется нулями.
При декодировании лишние нули не читаются, потому что известен originalLength.
